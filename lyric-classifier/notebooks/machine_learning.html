Preparando os dados para treinamento do modelo¶

Primeiro, para a representao do espao dimensional, se utilizar o conceito de Bag of Words, onde cada letra de msica passar por um processo de 'vetorizao', onde as dimenses deste vetor ser cada termo presente no documento. Para atribuir valores a cada dimeenso deste vetor resultante, ir ser adotada a mtrica de frequncia de termos TFIDF (conforme explicado no notebook 'data_analysis').

Ser criado um objeto 'vetorizador', apoiado a funo TfidfVectorizer (presente em sklearn.feature_extraction.text), para calcular o vetor TFIDF para cada letra de msica presente no dataset:

In [4]:
# Term Frequency Inverse Document Freq.(TF - Normalize Vectors, IDF: Penalize Frequent Words)
from sklearn.feature_extraction.text import TfidfVectorizer

# Objeto vetorizador
vectorizer = TfidfVectorizer(
                encoding   = 'utf-8',
                stop_words = stopwords,
                tokenizer  = tokenizer,
                min_df = 3,
                max_df = 0.6,
                sublinear_tf = True,
                ngram_range=(1, 2)
            ).fit(df['lyric'].values)

# Tabluando dataset
X = vectorizer.transform(df['lyric'].values) # Transforming
y = df['genre'].values

As configuraes do vetorizador seguiram como:

  • encoding = 'utf-8' (definimos o enconding para lidar com caracteres especiais, uma vez que so msicas brasileiras escritas em portuguss, em geral);
  • stop_words = stopwords (utilizaremos o arquivo 'stopwords' para eliminar as palavras pouco informativas - eg. 'a', 'e', 'de', etc.);
  • tokenizer = tokenizer (definimos uma funo que ser responsvel por 'quebrar' a letra de msica em vrias palavras; a vantagem de definir uma funo que podemos fazer alguns tratamentos no momento ANTES de calcular o TFIDF como normalizar as palavras (colocar todas em minsculas, por exemplo);
  • min_df = 3 ( o numro mnimo de documentos que um termo deve ser citado para estar presente no vetor resultante; caso contrrio o consideraremos como uma 'stopword', pois termos populares no so muito teis para o modelo);
  • max_df = 0.6 (ignoraremos termos que aparecem em mais de 60% dos documentos);
  • sublinear_tf = True (utilizamos a forma logartma da frequncia calculada - podenrando assim na mesma escala documentos com tamanhos diferentes);
  • ngram_range = (1, 2) (iremos considerar palavras isoladas e tambm bi-gramas, ou seja, conjunto de 2 palavras em sequncia na modelagem de TFIDF).

Por fim, com os dados preparados, dividiremos inicialmente o conjunto de dados entre treinamento e teste/validao. Esta diviso, em conjunto com outras estratgias (como a validao cruzada), permite avaliar o modelo calibrado diante de dados os quais o mesmo 'no conhece', viabilizando mensurarmos o vis e a varincia do modelo diante de uma mtrica de avaliao em tempo de treinamento e teste, medidas as quais mostram se o modelo est com overfitting (sobreajuste) ou underfitting (subajuste).

Classificador (Modelo ML)¶

Naive Bayes (Multinomial)¶

Conforme discutido brevemete, o modelo de ML mais natural para este domnio de problema o Nave-Bayes (NB). Tal escolha se justifica pois, alm de sua simplicidade de implementao e eficincia de operao, sua nica suposio sobre os dados a independncia entre todos os atributos e usa o teorema de Bayes para estimar a probabilidade de que as letras de uma msica pertenam a uma determinada classe, usando suas probabilidades anteriores.

O modelo NB comumente usado em problemas de minerao de texto, como a classificao de spam em e-mail, por exemplo, o que viabiliza adotarmos esta abordagem como baseline de nossa soluo.

Em tempo, como o NB um modelo probabilstico, este sofre com o caso de um termo no estar presente no corpus, resultando em P(g | t) = 0. Portanto, ser considerado termo alpha como fator de alisamento de cada atributo resultante no vetor TFIDF.

from sklearn.model_selection import GridSearchCV
from sklearn.model_selection import learning_curve
def plot_learning_curve(estimator, title, X, y, ylim=None, cv=None,
                        train_sizes=np.linspace(.1, 1.0, 5)):
    """
    Generate a simple plot of the test and traning learning curve.

    Parameters
    ----------
    estimator : object type that implements the "fit" and "predict" methods
        An object of that type which is cloned for each validation.

    title : string
        Title for the chart.

    X : array-like, shape (n_samples, n_features)
        Training vector, where n_samples is the number of samples and
        n_features is the number of features.

    y : array-like, shape (n_samples) or (n_samples, n_features), optional
        Target relative to X for classification or regression;
        None for unsupervised learning.

    ylim : tuple, shape (ymin, ymax), optional
        Defines minimum and maximum yvalues plotted.

    cv : integer, cross-validation generator, optional
        If an integer is passed, it is the number of folds (defaults to 3).
        Specific cross-validation objects can be passed, see
        sklearn.cross_validation module for the list of possible objects
    """
    
    #plt.figure()
    fig = plt.figure(figsize=(10,8))
    train_sizes, train_scores, test_scores = learning_curve(
        estimator, X, y, cv=5, n_jobs=1, train_sizes=train_sizes)
    train_scores_mean = np.mean(train_scores, axis=1)
    train_scores_std = np.std(train_scores, axis=1)
    test_scores_mean = np.mean(test_scores, axis=1)
    test_scores_std = np.std(test_scores, axis=1)

    plt.fill_between(train_sizes, train_scores_mean - train_scores_std,
                     train_scores_mean + train_scores_std, alpha=0.1,
                     color="r")
    plt.fill_between(train_sizes, test_scores_mean - test_scores_std,
                     test_scores_mean + test_scores_std, alpha=0.1, color="g")
    plt.plot(train_sizes, train_scores_mean, 'o-', color="r",
             label="Training score")
    plt.plot(train_sizes, test_scores_mean, 'o-', color="g",
             label="Cross-validation score")

    plt.xlabel("Amostras")
    plt.ylabel("Acurcia")
    plt.legend(loc="best")
    plt.grid(True) 
    if ylim:
        plt.ylim(ylim)
    plt.title(title)

Podemos ir ainda mais a fundo na avaliao de nosso modelo: dada esta matriz de confuso, podemos calcular outras mtricas como preciso, recall e f-score, que revelam outros aspectos de desempenho que so extremamente teis na avaliao de um modelo de ml. Seguem uma breve descrio de cada uma destas mtricas:

  • Precisso: apesar de parecido com a acurcia, a Preciso se refere a habilidade do classificador de NO rotular uma amostra negativa como sendo positiva. Para ficar mais claro, em nosso contexto, a habilidade de no classificar uma msica 'funk' sendo que ela do gnero 'bossa nova';

  • Recall: mensura a habilidade do classificador de encontrar os verdadeiros-positivos, ou seja, de classificar uma msica do gnero X como sendo de fato do Gnero X;

  • F-score: a mdia harmnica entre Preciso e Recall.

Para calcular estas mtricas utilizaremos a funo precision_recall_fscore_support distribudo em sklearn.metrics, conforme abaixo.

Conclusão¶

O modelo probabilstico de Nave Bayes se mostra eficiente para a resoluo do problema de classificao de gnero musical baseado em letras, com uma taxa de preciso de mais de 90% obtido pelo classificador proposto! Conforme os testes destacaram, corroborando a liteturatura correlata, o processo de preparao e tratamento dos dados possui grande influncia no resultado final do modelo.

Para a evoluo do modelo, podemos discutir os seguintes tpicos:

  • Uma vez que extraimos o vetor de palavras atrav do TFIDF, uma etapa potencialmente interessante para a melhoria do modelo seria a seleo de atributos mais significativos, uma vez que vrios termos podem no ter valores TFIDF que no so preditivos e, no geral, acabam inserindo vis ou confuso no modelo. Alm disso, com um menor espao de caracterstica, o desemepnho computacional poderia ser fortemente otimizado, dado que o processo de vetorizao via TFIDF pode resultar em um problema de grande escala dimensional;

  • Avaliar modelos de machine learning mais complexos para este domnio de problema, realizando um benchmarking mais completo;

  • A explorao adicional da preparao do conjunto de dados pode ser mais profundamente investigada. Por exemplo, seria interessante buscar padres sintticos que distinguem os gneros;

  • Avaliar outras abordagens no processo de criao de tokens, considerando outros n-gramas ou aplicar reduo do radical (stemizao) aos termos do conjunto de dados.

  • Avaliao de novos modelos que relevem o contexto semntico das composies, tal como Word2Vec.

Em linhas gerais, estas estratgias podem colaborar para melhorar ainda mais os resultados obtidos no problema de clssificao de gnero musical baseado em letras.
